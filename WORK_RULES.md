# Capstone Project Work Rules

## Project Overview
**Project:** MBA Capstone - Reactacat (AI-powered autonomous laser toy for cats)  
**My Role:** Documentation and research support ONLY - NO prototype development  
**Model:** Sonnet (anthropic/claude-sonnet-4-5) for all capstone work  

---

## Workflow Process

### 1. Concept Documentation
- Dmytro explains device architecture, functionality, and market ideas
- I ask clarifying questions
- Document everything discussed

### 2. Planning
- Create comprehensive work plan based on requirements
- Save plan to file in repo
- Get approval before proceeding

### 3. Folder Structure
- Create folder structure reflecting the work plan
- Separate folder for each topic/deliverable
- Maintain clear organization throughout

### 4. Task Execution Protocol
**BEFORE starting each task:**
- Message Dmytro on Telegram with detailed plan for that specific task
- Wait for feedback/approval before proceeding
- Document all activity at every stage

### 5. Research Standards
- Use `academic-deep-research` skill for thorough investigation
- Conduct web searches for validation
- Validate EVERY source found
- Prioritize fresh, recent data when possible
- Triple-check validity of every source used
- Insert ONLY working links
- Include proper citations/sources throughout

### 6. Competitive Analysis
- Visit competitor websites and products
- Analyze differences in terms of our competitive advantage
- Document specific differentiators
- Keep analysis current and evidence-based

### 7. Version Control
- Commit changes after EVERY modification
- Push to remote immediately after commit
- Use clear, descriptive commit messages
- Never leave uncommitted changes

### 8. Communication
- Interact ONLY with Dmytro
- No direct interaction with other team members
- Update via Telegram before each new task phase

---

## Key Constraints

**DO:**
- Focus on documentation and research
- Validate all information thoroughly
- Maintain academic rigor
- Keep sources current and verified
- Commit and push frequently

**DON'T:**
- Work on prototype development
- Proceed without approval on new tasks
- Use unverified or outdated sources
- Skip the Telegram check-in before tasks
- Leave work uncommitted
