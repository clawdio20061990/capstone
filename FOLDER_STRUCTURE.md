## File Naming Conventions

### Markdown Files
- Use lowercase with hyphens: `market-analysis.md`
- Date prefix for versions: `2026-03-07-market-analysis.md`
- Use descriptive names: `petcube-competitor-analysis.md` not `comp1.md`

### Data Files
- Include date: `survey-results-2026-05-15.csv`
- Include source: `bom-costs-digikey-2026-03.xlsx`

### Images/Charts
- Descriptive names: `market-growth-chart-2020-2026.png`
- Include figure numbers: `fig-3-1-competitor-matrix.png`

## Version Control

### Git Workflow
1. Commit after every significant change
2. Push to remote immediately after commit
3. Use clear commit messages:
   - `Add market analysis research findings`
   - `Update pricing strategy with competitor data`
   - `Complete Chapter 3 - Methodology`

### Backup Strategy
- All work in Git repository (GitHub)
- Regular commits ensure no work loss
- Critical deadlines: create tagged releases
